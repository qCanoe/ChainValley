# ChainValley (ChainVille) Phase Roadmap — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver the four delivery phases from `Project Description.md` (§6): on-chain fishery + MUD schema (Phase 1), Python orchestrator on local Anvil (Phase 2), batched experiment + export (Phase 3), offline analysis + report (Phase 4).

**Architecture:** Layer 1 is a MUD monorepo (`packages/contracts` + config) with a single `FisherySystem` enforcing soft vs hard quota; Layer 2 is the existing `chainvalley/` Python package extended with orchestration, prompts, and JSON-shaped LLM outputs; Layer 3 adds deterministic run IDs, condition flags, and append-only run logs; Layer 4 is Jupyter + Mann–Whitney U per §4.4.

**Tech Stack:** MUD 2.x (TypeScript config + Solidity systems), Foundry/Forge tests, Anvil local chain, Python 3.11+, OpenAI-compatible client (OpenRouter), `pytest`, Jupyter.
